                                          # Git

                                          # git commit

                                          将工作区中修改的文件提交到版本库

                                          # git branch [branch_name]

                                          创建一个新分支

                                          # git merge [branch_name]

                                          合并两个分支

                                          # git rebase [branch_name]

                                          合并分支, 和merge不同的是, rebase是取出一系列记录并复制到另一个地方, 这种做法会使git提交历史记录更加清晰

                                          # cat .git/HEAD

                                          查看HEAD指向

                                          # git symbolic-ref HEAD

                                          如果HEAD指向的是一个引用, 可以使用此命令查看指向

                                          # git checkout [commit_hash_id]

                                          更改HEAD的指向

                                          # git checkout [branch_name]^

                                          ^: 为相对引用的一个语法
                                          切换HEAD的指向为[branch_name]的父节点
                                          ~<num>: 可以一次移动很多步

                                          # git branch -f master [HEAD|branch_name|commit_hash_id]~<num>

                                          更改分支的指向

                                          # git reset [commit_hash_id]

                                          回退到某一次提交

                                          这种方式对远程分支是无效的, 因此需要使用git revert指令

                                          # git revert [branch_name]

                                          revert命令会创建一个新的提交记录

                                          # git cherry-pick [commit_hash_id]...

                                          将一些提交复制到当前所在分支下, 和git rebase有异曲同工之妙

                                          # git rebase --interactive/-i [branch_name|HEAD|引用地址]

                                          交互式rebase
                                          适用于不清楚提交记录的哈希值但又想合并两个分支的情况
                                          此命令会打开一个UI窗口, 通过拖拽鼠标调整提交记录的顺序

                                          # git tag [tag_name] [commit_hash_id]

                                          一个标识, 永远指向某个提交记录, 不会随着提交而移动
                                          如果不指定提交记录则会使用HEAD的指向

                                          # git describe <ref>

                                          检索标签
                                          <ref>是提交记录的引用, 如果没有指定则使用HEAD

                                          输出形式为:<tag>_<numCommits>_g<hash> tag是离ref最近的标签, numCommits是ref与tag相差多少个提交记录, hash是给定的ref所表示的提交记录哈希值的前几位

                                          # git clone

                                          克隆远程分支

                                          # git fetch

                                          获取远程仓库的数据
                                          fetch只是将数据下载下来, 并不会修改本地的文件, 需要和merge命令一起使用才会更新文件

                                          # git pull

                                          相当于git fetch + git merge

                                          # git push

                                          将本地变更上传到远程仓库

                                          # git pull --rebase

                                          相当于git fetch + git rebase
