# Cors跨域问题

# 为什么会有跨域问题

跨域问题是因为浏览器的安全机制，同源策略的限制

同源策略会阻止一个域的JavaScript脚本和另一个域的内容进行交互

所谓同源就是两个请求必须具备同样的协议、主机、端口号

请求url 被请求url 是否跨域 原因
http://naihe.com:8080/ http://naihe.com:8080/list 否 同源
http://naihe.com:8080/ https://naihe.com:8080/list 是 协议不同
http://young.com:8080/ http://naihe.com:8080/list 是 域名不同
http://naihe.com:8080/ http://naihe.com:8088/list 是 端口号不同

请求实际上是过来了 只不过前端没有读取响应

# 解决方案（后端）

# 使用@CrossOrigin注解

@RestController
public class ListController {

    @CrossOrigin
    @RequestMapping("/list")
    public List<String> list() {
        List<String> list = new ArrayList<>();
        list.add("Java");
        list.add("Python");
        list.add("C#");
        return list;
    }
}

# 全局方式

# 添加过滤器

@Configuration
public class CorsConfig {

    @Bean
    public CorsFilter corsFilter() {
        CorsConfiguration corsConfiguration = new CorsConfiguration();
        // 允许请求的域
        corsConfiguration.addAllowedOrigin("*");
        // 允许请求的头
        corsConfiguration.addAllowedHeader("*");
        // 允许请求的方式
        corsConfiguration.addAllowedMethod("*");
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        // 设置过滤请求路径
        source.registerCorsConfiguration("/**", corsConfiguration);
        return new CorsFilter(source);
    }
}

# 实现WebMvcConfigurer接口

@Configuration
public class CorsConfig2 implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        // 设置处理请求路径
        registry.addMapping("/**")
                // 允许的请求域
                .allowedOriginPatterns("*")
                // 允许的请求头
                .allowedHeaders("*")
                // 允许请求的方式
                .allowedMethods("GET", "POST", "DELETE", "PUT")
                // 是否允许携带cookie
                .allowCredentials(true)
                // 设置有效期 */s    3600秒内浏览器不必在询问
                .maxAge(3600);
    }
}
